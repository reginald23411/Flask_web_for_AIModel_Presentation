Project Objective

Apply what we have learned in class to build a data model that can handle data collection, cleaning, and classification of job advertisements.

Project Background

VTC conducts a regular manpower survey every four years, collecting information on the number of employees, income, education, etc. The project aims to assist the Government in developing industry manpower and training programs.

Business Requirement

We were asked to retrieve, clean, and classify 4 million unstructured job advertisements growing 100k to 150k per month into corresponding Principal Job Titles. The objective is to improve the efficiency and accuracy of job advertisement classification and enhance the operational efficiency of the manpower survey conducted by VTC HQIP, including enhanced data collection, cleaning, modeling, and classification of job advertisements.

Prediction Model: Random Forest

  1. Use the attributes (sales) to predict the class (error).
  2. A root node is the topmost decision node in a tree, usually the best predictor.
  3. An internal node is a test on an attribute. At each node, one attribute is chosen to split training examples into distinct classes as much as possible.
  4. A branch represents an outcome of the test, e.g., color=red.
  5. A leaf node represents a class label or class label distribution.

Advantages:

  • Inexpensive to construct
  • Extremely fast to use
  • Easy to interpret for small-sized trees
  • Accuracy comparable to other classification techniques for many simple data sets

Disadvantages:

  • Easy to overfit
  • Decision boundary restricted to being parallel to attribute axes
  • Large tree is difficult to interpret
